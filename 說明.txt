//文件結構 HTML 概述 ==============================================================

	01. 此平台 HTML 由'群組'為最小單位堆疊而成，具有'data-type'屬性的區塊即是'群組'
	01. '群組'分為輸出內容的'模組群組(模組)'、 做完排版容器的'排版群組(群組)'及模擬模組的群組'排版模組群組(模組群組)'
	01. '群組'分為輸出內容的'模組群組(模組)'、 做完排版容器的'排版群組(群組)'及模擬模組的群組'排版模組群組(模組群組)'
	02. '模組群組'呈現特定內容，如'RSS模組'或'天氣模組'等等
	03. '排版群組'作為排版容器，裝載其他'群組'以達到排版所需
	04. '排版模組群組'可將多個'群組'組成一個'模組群組'，具有'.header'與'.content'區塊，以清單方式排列內部'群組'( ul>li>'群組')
	05. '框架性 排版群組'為框架基礎，具有 id 屬性，不可拖曳、不可變動。 具體成員有'#root'、'#mobile'、'#extend'、'#wrapper'、'#header'、'#content'、'#footer'、'#index'、'#aside'、'#section'、'#article' 等十一項
	06. '框架性 排版群組'的'.header'、'.content'、'.footer'均為'排版群組'的一種 ，有別於一般'模組群組'內的同名區塊
	07. '模組群組'的'.header'、'.content'與'.footer'僅作為一般內容使用，詳見'//模組群組 HTML 規範'
	08. 以下為框架結構：

		html

			- head

			- body

				- form

					- .sys-root //拖曳根節點

						- .base-mobile //行動載具

						- .base-extend //額外資訊

						- .base-wrapper //固定佈局

							- .base-header //頁首資訊

							- .base-content //主要內容

								- .base-module-area //模組區塊

								- .base-page-area //內頁區塊

									- .base-aside //內頁側欄區塊

									- .base-section //內頁內容區塊

											- .base-article //文章

							- .base-footer //頁尾資訊

	09. 


//自訂屬性 HTML 規範 ===========================================================

	01. 'data-*'參數定義該節點的狀態或呈現更多資訊
	01. 以下為'data-*'參數列表：
		01-02. data-function		用於'群組'，啟用的 function 名稱，屬性為字串
		01-02. data-functionData	用於'群組'，賦予啟用 function 各項參數，以 json 方式匯入
		01-05. *data-type			用於'群組'，模組型態，屬性為數字，'0'為'模組'、'1'為'分割'、'2'為'頁籤'、'3'為'單欄'、'4'為'模組群組'
		01-06. *data-index			用於'群組'，模組順序，屬性為數字，如'1'代表父層的第一個子節點，特殊模組如頁籤、導盲無此屬性
		01-07. *data-child			用於'群組'，模組子節點數量，屬性為數字，如'1'代表擁有1個子節點，寬度預設為並排
		01-08. data-aLineLength		用於'群組'，模組一行內的'群組'並排數量，屬性為數字，如'3'代表一行有三個'群組'並排，會覆蓋 data-index 的排序規則
		01-01. data-js				用於'body'，判斷是否開啟 js ，屬性為布林值'false'&'true'
		01-03. data-admin			用於'body'，判斷是為管理員，屬性為布林值'false'&'true'
		01-09. data-navWidth		用於'.nav'，判斷次選單寬度，屬性為數字，如'2'代表子選單為第一層選單的 200% 的寬度

// .header 模組標題 HTML 規範 ===========================================================

	01. '.header'放置模組標題，必須為文字
	02. 若模組本身就是標題，如'標題名稱模組(.hgroup)'或'文章標題模組(.title)'等，應省略'.header'
	02. 使用'標題組件'時，模組標題應使用 h4
	03. 特定模組標題並非 h4 標籤，'標題名稱模組(.hgroup)'及'排版模組群組'標題為'h3'、'商標模組(.logo)'標題為'h2'
	05. '框架性 排版群組'的 .header 非模組標題，詳見'//文件結構 HTML 概述'
	06. 


// .content 模組內容 HTML 規範 ===========================================================

	01. '.content'裝載主要的模組內容
	02. 若模組本身就是標題，如'標題名稱模組(.hgroup)'或'文章標題模組(.title)'等，'.content'應裝載'標題組件'並省略 .header
	03. '框架性 排版群組'的'.content'非模組內容，詳見'//文件結構 HTML 概述'
	04. 


// .footer 模組附加資訊 HTML 規範 ===========================================================

	01. .footer 放置附加資訊如按鈕、說明文字
	02. 按鈕類附加資訊，格式為'清單組件'，若需 class 名稱則加在 li 上
	03. 按鈕類附加資訊，基本按鈕 class 可用'.prev(上)'、'.next(下)'、'.first(第一)'、'.last(最後)'與'.more(更多)'，被啟用的狀態可用'.is_active(啟動)'
	04. 文字類附加資訊，格式為'內文組件'
	05. '框架性 排版群組'的 .footer 非模組附加資訊，詳見'//文件結構 HTML 概述'
	06.
